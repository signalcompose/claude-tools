# X Articles 記事構造ガイド

`draft` コマンドを使う際は、このファイルを**必ず最初に読む**こと。

---

## 記事の方針（厳守）

### 何を書くか

- ✅ **汎用ナレッジ共有** を目的とする（「〇〇とは」「〇〇の方法」など技術的知見）
- ❌ 特定プロジェクト・ツールの宣伝記事にしない

### プライバシー・公開範囲の判断

| 種類 | 扱い |
|------|------|
| プライベートリポジトリ名・プロジェクト名 | ❌ 伏せる（「あるOSSプロジェクト」「チームで使っているツール」等に置き換え） |
| 社内固有のコード・実装詳細 | ❌ 必ず抽象化 |
| GitHub public リポジトリ・OSS | ✅ そのまま記載可 |
| パブリックな技術（フレームワーク・言語）| ✅ そのまま記載可 |

### 文体・長さ

- **コンパクト**に書く（長文は読まれない。各セクション短め）
- 各段落 3〜4 文以内
- 読者が「何を検索すれば良いか」が伝わる書き方
- 正確さより**読者のきっかけ作り**を重視（完璧でなくてよい）

---

## 構成テンプレート

```
## For English Readers
（英語圏の読者向け要約、3〜4文。「This article is in Japanese. Here's a quick summary in English.」から始め、
Google翻訳への誘導を含める）

TL;DR（英語、3〜4文。日本語記事の内容を英語でまとめたもの）
※ TL;DR = Too Long; Didn't Read。英語圏読者向けの英語サマリー。日本語まとめではない。

---

## 本文セクション
H2 見出しで 2〜4 個。各セクション短め（300字程度を目安）。

## まとめ
比較・結論（テーブルではなくテキスト）。次のアクションを示す。

---
*参照リンク*（イタリック）
---
## Acknowledgement
（AI利用の開示など、必要な場合のみ追記）
```

## X Articles の制約（マークダウン変換ルール）

X Articles エディタは DraftJS ベースのため、一部のマークダウン要素は手動操作が必要。

### 自動変換可能

| マークダウン | X Articles 変換 |
|------------|----------------|
| `## 見出し` | `<h2>` → エディタの見出しブロック |
| `**太字**` | `<strong>` → ボールドテキスト |
| `*イタリック*` | `<em>` → イタリックテキスト |
| `- リスト` | `<ul><li>` → 箇条書き |
| `[テキスト](URL)` | `<a href="URL">` → リンク |
| `> 引用` | `<blockquote>` → 引用ブロック |

### 手動挿入が必要（フラグを挿入）

| マークダウン | 変換方法 |
|------------|---------|
| ` ``` コードブロック ``` ` | `[ここにコードを手動挿入: コードブロックボタンで追加]` |
| `---` 水平線 | `[ここに仕切りを手動挿入: エディタの「─」ボタンで追加]` |

### 変換が必要（テキスト変換）

| マークダウン | 変換方法 |
|------------|---------|
| `\| テーブル \|` | `Before → After` テキスト形式、または箇条書きで表現 |
| `![画像](url)` | 画像は別途ファイルアップロード UI で追加 |

## 記事の品質チェックリスト

ドラフト完成後に確認:

- [ ] プライベートなプロジェクト名・リポジトリ名が伏せられているか
- [ ] コードが抽象化されているか（プロジェクト固有の変数名等が含まれていないか）
- [ ] For English Readers セクションが英語で書かれているか
- [ ] TL;DR が英語で書かれているか
- [ ] 各セクションが短くコンパクトか
- [ ] テーブルがテキスト形式に変換されているか
- [ ] コードブロック・仕切りにフラグが挿入されているか
