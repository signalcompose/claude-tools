# Refactor Team

チーム編成によるコードリファクタリング。分析結果をユーザーに提示し、承認後に実行。

## 使い方

```
/code:refactor-team [対象パス]
```

対象パスを省略すると、ベースブランチとの差分ファイルを自動検出します。

## ワークフロー

1. **対象特定** — 対象ファイルの解決、プロジェクトコンテキスト検出
2. **リファクタチーム作成** — analyzer + refactorer を起動
3. **分析実行** — DRY違反・複雑度・命名・構造を分析、優先度付き提案を作成
4. **ユーザー承認** — 提案一覧を提示し、実行する項目をユーザーが選択
5. **リファクタリング実行** — 1リファクタ = 1コミット、テスト通過を確認
6. **完了報告** — 実施サマリー表示

## 制約

- 機能変更は禁止（リファクタリングのみ）
- 各リファクタリングは個別コミット（`refactor:` prefix）
- ユーザー承認なしの自動実行は禁止

## 関連

- **PRレビュー**: `/code:pr-review-team`（PR作成後のチームレビュー）
- **pre-commit レビュー**: `/code:review-commit`（コミット前の品質ゲート）
